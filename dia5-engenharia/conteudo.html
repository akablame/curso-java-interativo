O que é UML?

Unified Modeling Language
Analogia: Planta baixa de uma casa

Antes de construir uma casa, o arquiteto faz a planta baixa. UML é a "planta baixa" do software — você desenha o sistema antes de codar, economizando tempo e evitando retrabalho.

Diagrama de Casos de Uso

Quem faz o quê?

O diagrama de casos de uso responde: "Quem usa o sistema?"(atores) e "O que eles podem fazer?"(casos de uso). É a visão do usuário sobre o sistema.

Dica de identificação de atores

Para encontrar os atores, pergunte: "Quem vai usar este sistema?"e "Há sistemas externos que interagem?". Atores sempre ficam forado retângulo do sistema.

Diagrama de Sequência

Sequence Diagram
Analogia: Roteiro de uma ligação

O diagrama de sequência é como um roteiro: mostra quem fala com quem, o que dize em que ordem. As "linhas de vida" representam os participantes.

Fluxo de Login — Exemplo

Diagrama de Sequência: Login
Usuário──► Interface: digitarCredenciais(login, senha)
Interface──► SistemaLogin: autenticar(login, senha)

BancoDados◄── BancoDados: [retorna usuario ou null]
SistemaLogin──► SistemaLogin: verificarSenha(hash)
Interface◄── SistemaLogin: retornaResultado(sucesso/falha)
Usuário◄── Interface: exibirMensagem()

Java — Estrutura MVC
// MODEL (representa os dados)
public class Aluno {
 private int id;
 private String nome;
 private double nota;
 // getters, setters, construtores
}

// DAO (acessa o banco — parte do Model)
public class AlunoDAO {
 public void salvar(Aluno a) { /* INSERT no BD */ }
 public Aluno buscarPorId(int id) { /* SELECT */ }
 public List<Aluno> listarTodos() { /* SELECT * */ }
}

// CONTROLLER (regras de negócio)
public class AlunoController {
 private AlunoDAO dao = new AlunoDAO();

 public void cadastrarAluno(String nome, double nota) {
 if (nome.isEmpty()) throw new IllegalArgumentException("Nome vazio!");
 if (nota < 0 || nota >10) throw new IllegalArgumentException("Nota inválida!");
 Aluno a = new Aluno(nome, nota);
 dao.salvar(a);
 }
}

// VIEW (JFrame — só exibe e captura input)
public class TelaAlunos extends JFrame {
 private AlunoController controller = new AlunoController();

 void aoClicarSalvar() {
 String nome = txtNome.getText();
 double nota = Double.parseDouble(txtNota.getText());
 controller.cadastrarAluno(nome, nota); // usa o controller!
 JOptionPane.showMessageDialog(this, "Salvo!");
 }
}
Por que usar MVC?

Separação de responsabilidades: se mudar a tela (View), não precisa mudar o banco (Model). Se mudar a regra (Controller), não precisa mexer na tela. Facilita manutenção e trabalho em equipe!

Scrum — Metodologia Ágil

Agile
Analogia: Entrega de pizza

Em vez de demorar 6 meses para entregar o sistema completo, o Scrum entrega partes funcionais a cada 2 semanas(Sprint). O cliente vê resultado rápido e pode dar feedback!

PapelResponsabilidade
Product Owner (PO)Define o que deve ser feito (backlog), prioriza funcionalidades
Scrum MasterFacilita o processo, remove impedimentos, protege o time
Dev TeamDesenvolve o produto — programadores, designers, testers

Cerimônias do Scrum

Planejamento do Sprint: time decide QUAIS tarefas do backlog serão feitas neste Sprint (1-4 semanas).
Reunião diária de 15 minutos: 1) O que fiz ontem? 2) O que farei hoje? 3) Há algum impedimento?
No final do Sprint, o time demonstra o que foi produzido para o Product Owner e stakeholders.
Time reflete sobre o Sprint: o que funcionou bem? O que pode melhorar? Gera plano de melhoria.
